simple Flask App | An Introduction to Flask

Introduction

Flask is a micro-web framework used for creating both small and enterprise level web applications.

Flask is considered a micro-web framework due to the fact that it does not rely on a particular set of tools or libraries. Due to this Flask has developed a great community around it and has thousands of different tools and libraries that you can make use of in your projects and you only call on them when you have to.

Flask was written / developed in python without a database abstraction layer, making it lightweight and easy to use compared to one of it's closest buddies Django which is better suited for large-scale database-driven web applications.

Flask is great and easy to learn as it's one of the languages that really don't require much to set up and for now all we will need is quite a simple file structure that we will move onto creating in the first steps towards your first flask-App.

For now, we will just be looking into Flask.

Setup

the first thing we need to do is make sure that we install Flask, to do this I am documenting the code for terminal commands as linux based commands, just an FYI you may have to alter these slightly if you are using an OS different from linux.

If you have experience or worked with web applications before you may have noticed that some apps have certain file structures that are defined and expected to be followed to help the process of finding and locating the correct files that they need.

Flask is no different.

Infact with Flask it is not essential to use the filestructure but for the sake of simplicity we are going to be using a basic file structure, especially if you are knew to Flask this will help get your head round some of the basic features to see how the Flask framework runs.

So first off we are going to need to install some packages that will come in useful when we are working with the Flask framework.

From here on we are assuming that you have python installed already, if not please head here for a guide on installing python.

For starting this project please create a folder called "flask-app" and open this folder as this is the one we will be storing all our files for the project in.

Installing Pip

Pip is a package manager for python that we are going to be using to install different packages or modules that we need to use in our project, including installing Flask as it can download the flask module quickly with all the packages we need.



    

  1. kcwebdev:~/flask-app$ curl https://bootstrap.pypa.io/get-pip.py -o get-pip.py
    2. 

  2. kcwebdev:~/flask-app$ python get-pip.py|
    3.

Installing virtualenv

virtualenv is a tool that we are going to use for creating an isolated python environment. This means that we can work on this project without affecting any of our other projects in the future. 



    

  1. kcwebdev:~/flask-app$ pip install virtualenv
    2. 

  2. kcwebdev:~/flask-app$ virtualenv .venv
    3. 

  3. kcwebdev:~/flask-app$ source .venv/bin/activate
    4. 

  4. (.venv)kcwebdev:~/flask-app$|
    5.

Above you can see the steps taken to install virtualenv, you might have noticed a few things happening, all that has happened is a new folder has been set up called .venv and notice how on the command line next to our username it says"(.venv)" this indicates that the environment is active.

To stop using this virtual environment all you need to do is type deactivate.

Installing Flask



    

  1. (.venv)kcwebdev:~/flask-app$ pip install Flask|
    2.

Now remember when I was discussing the file structure and how it's going to help us so for the next step we will create the neccessary folders that will make working with Flask a whole lot easier!



    

  1. (.venv)kcwebdev:~/flask-app$ mkdir static templates|
    2.

All we are doing in the above command is creating two new folders; static and templates. The static folder will be used in the future and is where we will store all our Images, CSS and JS files. The templates folder wil be used sooner than the static folder as this is the one that we will be adding any re-usable components into that we code using html in the near future.

Basic code

Now that the main structure is in place we can focus on getting something to look at on the screen!

To do this we need to create the main python file that will contain the code to run the Flask app, in this file we will be able to import modules / packages we installed using pip and reference them for use in our code, we will call this file "app.py", to do this we can run the following command: 



    

  1. (.venv)kcwebdev:~/flask-app$ touch app.py|
    2.

Once you have created the file open it up and add in the following code to get our Flask app up and running!



    


  1. from flask import Flask
    2. 

  2. app = Flask(__name__)
    3. 

  3. 

  4. @app.route("/")
    5. 

  5. def hello():
    6. 

  6.   return "hello World!"
    7. 

  7. 

  8. if __name__ == "__main__":
    9. 

  9.   app.run()
    10.

Now save that file as app.py and jump back over to the command line and run the following command



    

  1. (.venv)kcwebdev:~/flask-app$ python app.py|
    2.

Output should be similar to this: 

 * Serving Flask app "app" (lazy loading)
 * Environment: production
   WARNING: Do not use the development server in a production environment.
   Use a production WSGI server instead.
 * Debug mode: off
 * Running on http://127.0.0.1:5000/ (Press CTRL+C to quit)

Now open up a web browser and enter "http://127.0.0.1:5000" in the URL bar and hit search to see the page you've created.

Boom! now you can see your first Flask app with a Hello World! message displayed to the user.
