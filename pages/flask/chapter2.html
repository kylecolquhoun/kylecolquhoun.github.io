Routes

In the code for the first example you seen that we made use of routes already, if you didn't notice, we done this by declaring the @app.route("/") decorator. In the decorator we passed the value of ""/". This just means that when the user types the forward slash after the url, in this case localhost:5000/, that it should look for code that has been defined in the function assigned to the decorator.

Having a meaningful URL name comes with it's benefits, it creates a readable more user-friendly layout that users can easily memorise, by doing this we can also improve SEO for our projects; this in turn improves our sites ranking.

Templates

Remember in the first chapter how we set up the templates folder? well we are now going to start making use of it.

Flask makes use of the jinja template library to render templates on the users browser, templates are just files that contain static data that can be re used, they can also have placeholders / variables in them that allow us to dynamically render content for the user. We will be using HTML to create our templates.

I know it looks like theirs alot going on above but lets break it down again.

  • Line 1: {% extends 'base.html' %} this tells Flask that this template will inherit the content from base.html, this is good as it means we dont need to add in the whole html page structure in our template file as you can see we left out the common html, head, title tags etc as they are no longer neccessary.
  • Line 2 - 4: {% title %}Planes{% endblock %} remember this from the base.html file? the only difference this time round is that we have added in a value between the opening and ending block, this value is what we want the title of the page to be, in this case it would be Planes.
  • Line 5 - 7: This works the same as the last block, however their is one difference we have passed a piece of html code as the value. If you look at the "src" attribute on the "img" tag, you will see we have used another great feature that Flask offers. {{ url_for('static', filename='images/plane.jpg')}} This is telling the app to look in the images directory that is inside our static directory, and find an image with the name plane.jpg. This is useful as it saves us having to provide a complete path to the image and if we decided to change the structure in the future, their wouldn't be much bother as the app still knows to look for the file and folders provided.
  • Line 9 - 11: Finally! These lines of code also make use of a block, they do this so that we can provide some information about planes to show the user on the webpage.
